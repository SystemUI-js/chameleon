# 新增 CDock 组件

## TL;DR

> **Summary**: 在现有 React + TypeScript 组件库中新增 `CDock`，沿用 `CWidget` 继承模式，实现四边吸附布局、受控/默认值 API 与静态样式契约；本期不实现任何拖拽、缩放或内部位置切换交互。

## Context

### Original Request

新增 `CDock` 组件，继承自 `CWidget`，一般出现在 `Screen` 组件中，支持吸附 `top | right | bottom | left`。Props 包含 `position`、`onPositionChange`、`height`、`onHeightChange`、`gapStart`、`gapEnd`，例如 `position="bottom"` 时应表现为 `left: gapStart`、`bottom: 0`、`right: gapEnd`。

### Interview Summary

- 已确认 API 采用“受控 + 默认值”模式。
- 已确认本期不实现内建交互；`onPositionChange` / `onHeightChange` 仅作为预留接口，不要求本期主动触发。
- 已确认测试策略为 TDD。
- 已确认本期验收口径为“保留 API 即可”，不为两个回调补临时内部触发入口。

### Metis Review (gaps addressed)

- 明确 `height` 在四个方向都表示 Dock 厚度；当 `position` 为 `left` / `right` 时，它映射为视觉宽度。
- 明确 `gapStart` / `gapEnd` 只接受 `number`，语义均为像素值。
- 明确默认值：`gapStart=0`、`gapEnd=0`、`defaultPosition ?? 'top'`。
- 明确本期 guardrail：禁止拖拽吸附、禁止 resize handle、禁止 DockManager、禁止多 Dock 编排。
- 明确验收重点：四边样式映射、受控/默认值解析、回调静默、导出可用、lint/test/build 通过。

## Work Objectives

### Core Objective

新增一个可复用的 `CDock` 组件，使执行者无需额外判断即可按既有类组件模式实现四边吸附布局，并通过自动化测试锁定 API、样式映射与导出行为。

### Deliverables

- 新增 `src/components/Dock/Dock.tsx`，导出 `DockPosition` 类型与 `CDockProps` / `CDock`。
- 新增 `src/components/Dock/index.scss`，提供 `cm-dock` 与方向修饰类。
- 更新 `src/components/index.ts`，将 `CDock` 纳入组件 barrel。

- 新增 `tests/Dock.test.tsx`，覆盖受控/默认值、四边样式、样式合并、回调静默、导出可用。

### Definition of Done (verifiable conditions with commands)

- `yarn lint` 通过。

### Must Have

- `CDock` 继承 `CWidget`，而不是新建独立函数组件体系。
- `position` 支持 `top | right | bottom | left`。
- 组件支持 `position` / `defaultPosition` 与 `height` / `defaultHeight` 的受控+默认值口径。
- `gapStart` / `gapEnd` 以像素数值参与布局计算。
- 四个方向的样式映射完全明确且可测试。
- `children` 透传渲染。
- `className` / `style` / `data-testid` 可用，且不破坏核心吸附边。

### Must NOT Have (guardrails, AI slop patterns, scope boundaries)

- 不新增拖拽吸附、边缘缩放、动画、持久化、DockManager、上下文状态、碰撞解决、多 Dock 排布。
- 不修改 `Screen` 的 `CGrid` 结构，也不要求 `Screen` 注入额外 props。
- 不引入 `any`、不新增默认导出、不切换为 CSS-in-JS。
- 不允许把调用方 `style` 静默覆盖核心边定位规则。
- 不允许为回调补“伪交互”逻辑，只为了让测试触发。
