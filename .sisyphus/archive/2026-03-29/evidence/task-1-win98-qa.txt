## Architecture Notes

### Harness Route System
The common controls harness now supports two routing modes:

1. **Fixture route** (`?fixture=default|disabled`): Renders controls directly without theme context
2. **System-theme route** (`?systemType=windows&theme=win98`): Renders controls with Win98 theme classes applied

### Theme Application
Unlike `windowHarness` which uses `DevSystemRoot` to wrap content in the full Windows UI shell (Window + StartBar), the common controls harness applies theme classes directly to a container div:

```tsx
<div className={`${SYSTEM_CLASS_NAMES[systemType]} ${THEME_CLASS_NAMES[theme]}`}>
  <HarnessLayout>
    {/* Controls */}
  </HarnessLayout>
</div>
```

This allows testing controls in isolation without the Window shell while still applying Win98 theme CSS.
