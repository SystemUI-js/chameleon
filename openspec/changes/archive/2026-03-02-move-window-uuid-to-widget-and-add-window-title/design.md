## Context

当前窗口体系中，`CWindow` 直接持有 `uuid`，而 `CWidget` 作为所有窗口/控件的基类尚未统一承担实例身份职责。这导致身份管理分散，不利于后续窗口管理、拖拽交互和跨主题扩展。与此同时，现有窗口实现没有可复用的标题栏基类，无法以“用户自由组合”的方式在不同主题中复用标题栏与拖动行为。

本变更需要在不破坏现有组件组合模型的前提下，引入 `CWindowTitle` 基类（默认不自动挂载到 `CWindow`），并通过 `@system-ui-js/multi-drag` 实现“拖动标题栏即移动父窗口”的行为，最终在 Win98 主题落地一个基础标题栏实现进行验证。

约束：
- 保持组件库现有的 class-based widget/window 架构与导出风格。
- 不引入隐式子组件挂载，保持组合式 API 一致性。
- 不改变非窗口组件行为。

## Goals / Non-Goals

**Goals:**
- 将 `uuid` 身份属性上移到 `CWidget`，统一子类实例标识来源。
- 引入 `CWindowTitle` 基类组件，默认空内容、默认不被 `CWindow` 内置渲染。
- 通过标题栏作为 drag-handle 驱动所属 `CWindow` 位置更新。
- 在 Win98 主题中提供继承实现（基础标题栏样式/结构）并验证拖拽联动。

**Non-Goals:**
- 不在本变更中引入完整窗口管理器重构（如 z-index/焦点栈重设计）。
- 不在本变更中定义复杂标题栏按钮系统（最小可用标题栏即可）。
- 不在本变更中新增非 Win98 主题的完整适配。

## Decisions

1. **将 `uuid` 放入 `CWidget`（而非继续留在 `CWindow`）**
   - 方案 A（采用）：`CWidget` 初始化并暴露稳定 `uuid`，`CWindow` 继承使用。
   - 方案 B（未采用）：`CWindow` 保留 `uuid`，其他组件按需复制。
   - 选择理由：A 可以消除重复身份逻辑，符合“基类承载共性状态”原则，并为未来更多 widget 子类复用提供一致基础。

2. **`CWindowTitle` 设计为可组合基类，默认不由 `CWindow` 自动注入**
   - 方案 A（采用）：用户在使用层显式组合 `CWindowTitle`。
   - 方案 B（未采用）：`CWindow` 内置默认标题栏并允许覆盖。
   - 选择理由：A 与当前库的可组合设计更一致，避免内置 UI 造成主题与业务耦合；也更符合用户“主动自由组合”的预期。

3. **拖拽行为绑定在 `CWindowTitle`，位置写回 `CWindow`**
   - 方案 A（采用）：标题栏作为手柄，拖拽事件更新父窗口坐标。
   - 方案 B（未采用）：整窗可拖拽。
   - 选择理由：A 更符合桌面窗口交互习惯，并避免内容区域误触拖动。

4. **依赖 `@system-ui-js/multi-drag` 实现拖动交互**
   - 方案 A（采用）：复用现有依赖，通过姿态/位置更新回调驱动父窗口移动。
   - 方案 B（未采用）：新增其他拖拽库或自写 pointer 逻辑。
   - 选择理由：A 依赖稳定、与项目目标一致，能降低额外依赖和维护成本。

5. **Win98 主题采用“继承 + 基础样式”进行验证**
   - 方案 A（采用）：实现最小标题栏结构（文本区域 + 基本视觉风格），验证拖拽闭环。
   - 方案 B（未采用）：一次性实现完整复古标题栏细节。
   - 选择理由：A 可以先验证核心行为和组合契约，再增量完善视觉细节，降低交付风险。

## Risks / Trade-offs

- **[Risk] `uuid` 上移可能影响依赖 `CWindow.uuid` 初始化时机的逻辑** → **Mitigation**：保持 `CWidget` 构造期即生成 `uuid`，并补充回归测试覆盖窗口实例标识可用性。
- **[Risk] 标题栏拖拽事件与窗口内容交互冲突** → **Mitigation**：将拖拽触发域严格限制在 `CWindowTitle`，并在实现中隔离 pointer 事件边界。
- **[Risk] 不自动挂载标题栏会让迁移用户短期需要显式组合** → **Mitigation**：在文档和示例中明确组合方式，并提供 Win98 主题示例。
- **[Trade-off] 先做基础标题栏，视觉细节不完整** → **Mitigation**：将细节样式增强留到后续迭代，不阻塞核心交互上线。

## Migration Plan

1. 在 `CWidget` 增加 `uuid`，保留 `CWindow` 对外实例可读 `uuid` 行为（通过继承保持兼容）。
2. 引入 `CWindowTitle` 基类及基础导出，不在 `CWindow` 默认渲染树中挂载。
3. 在 Win98 主题新增/调整标题栏实现，接入拖拽并完成联动。
4. 增加测试覆盖：
   - `uuid` 由基类提供；
   - 标题栏拖拽驱动窗口位置变化；
   - 非标题栏区域不触发拖动。
5. 若出现回归，回滚步骤：
   - 临时恢复 `CWindow` 本地 `uuid` 定义；
   - 关闭标题栏拖拽绑定；
   - 保留组件接口但停用新行为。
