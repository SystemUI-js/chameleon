
- CSelect now uses CMenu for the visible interaction and keeps the external `data-testid` on the visible button trigger. The internal native select uses `${data-testid}__native` when an external id exists, otherwise `cm-select-native-control`.
- CMenu has no root disabled prop, so CSelect prevents opening by disabling the trigger button and also guarding the trigger click handler with `event.preventDefault()`.
- Hidden native select remains in the form with `name`, `value`, `disabled`, and `required`; placeholder selection is represented as an empty disabled option so `required` remains invalid until a real option is selected.
- Uncontrolled form reset needs explicit React state reset because the visible trigger is no longer the native control; CSelect listens to the parent form `reset` event and restores the initial default/placeholder value.
